AttributeGraphs.AttributeGraph — Type
struct AttributeGraph{T<:Integer, G<:AbstractGraph{T<:Integer}, V, E, R} <: AttributeGraphs.AbstractAttributeGraph{T<:Integer, G<:AbstractGraph{T<:Integer}}

  • graph::AbstractGraph{T} where T<:Integer: the wrapped graph

  • vertex_attr::Any: the attributes for the vertices

  • edge_attr::Any: the attributes for the edges

  • graph_attr::Any: the attributes for the graph

If the type for the vertices attributes and the edges attributes is a AbstractVector then there is a default behavior.

source
AttributeGraphs.AttributeGraph — Method
AttributeGraph(
    gr::AbstractGraph{T<:Integer};
    vertex_type,
    edge_type,
    graph_type
) -> AttributeGraph{T, _B, Missing, Missing, Missing} where {T<:Integer, _B<:AbstractGraph{T}}

Construct a AttributeGraph.

Keywords

  • vertex_attr_type: The element type for the vertex attributes
  • edge_attr_type: The element type for the edge attributes
  • graph_attr_type: The element type for the graph attributes

All types need to be able to call an empty constructor.
